> [!Important]
> Extensive refactoring work was introduced in this update. If you encounter any issue during installation or update, please uninstall and re-install Daily Routines.

## Framework Modifications

- New `Device Credential` has been introduced. Compared to the existing Machine Credential (MID), TID is built on a more stable and modern unique identification algorithm, making it significantly less likely to change across most scenarios. Please note that this feature is currently in early beta. We welcome any feedback to help us further refine and perfect the algorithm!
- The link to module feedback has been updated.
- The compiling and loading logic of modules have been optimized.
- The logic of Interrupt Hotkey has been adjusted. A re-configuration is required after this update.
- An issue that the button on controllers may be omitted in the display text of Interrupt Hotkey has been fixed.
- The internal logic of localization has been adjusted. A re-configuration is required after this update.
- The button `Update Translations` as long as its function have been removed to prevent unexpected issues.
- The visual effect of card components in module configuration interfaces has been optimized.
- The appearance of prompts indicating that parameters are invalid has been adjusted.
- The localization files belong to `Traditional Chinese` have been adjusted. Now options `French` and `German` have been introduced, however these two options are just placeholder for now.
- The localization performance has been improved.
- The appearance of the `authentication` interface has been adjusted. More informative components have been introduced.
- The module search system has been refactored. The performance and command suggestions have been improved.

## Removed Modules

### Auto Prevent Duplicate Mudras [Action]

The feature has been fully merged into the module `Auto Ninjutsu` (AutoTenChiJin).

### All the 5 modules related to Glamour Dresser in [UI_Operation] and [UI_Optimization]

The features of these modules have been fully merged into an integrated module `Auto Manage Glamour Dresser` (AutoGlamourDresser).

## New Modules

### Direct Command Control For UI [Assistance]

*Recommended Module*

Provide commands in the format of /pdr {0} <​UI internal name​> <​parameter(s)​> for direct control and interaction with various in-game UIs.

![CallbackCommand](/assets/Changelog/2.0.3.0/CallbackCommand.png)

### Direct Command Access To Agent [Assistance]

*Recommended Module*

Provide commands in the format of /pdr {0} <​agent name​> <​event type​> <​parameter(s)​> for direct access to various internal agents.

![ReceiveEventCommand](/assets/Changelog/2.0.3.0/ReceiveEventCommand.png)

### Auto Replace Race And Gender [System]

Automatically apply a specified combination of race and gender on characters in certain combinations of race and gender after proper configurations.

![AutoChangeCharacterRaceSex](/assets/Changelog/2.0.3.0/AutoChangeCharacterRaceSex.png)

### Auto Manage Glamour Dresser [UI_Operation]

*Recommended Module*

Provide a series of life-quality features in Glamour Dresser, including automatically switching to the catalog corresponding to the current job, and retrieving gears which are duplicate, or share the same item model, or could be stored in The Armoire, or could be stored as an outfit glamour.

![AutoGlamourDresser](/assets/Changelog/2.0.3.0/AutoGlamourDresser.png)

### Customize Command Alias [System]

Apply customized aliases for in-game commands after proper configurations.

![CustomizeCommandAlias](/assets/Changelog/2.0.3.0/CustomizeCommandAlias.png)

## Module Modifications

### Better Search In Right-click Menu [UI_Optimization]

- The code has been completely refactored and streamlined.
- An issue of the incorrect link to `ERIONES` Korean website has been fixed.
- Now inquiries to [FFXIV Traditional Chinese Toolbox](https://cycleapple.github.io/ffxiv-item-search-tc) are supported.

### Auto Prevent Duplicate Actions [Action]

- A detection for `Tendo the Mirrored` during `Samurai`'s action `Meikyo Shisui` has been introduced.

### Auto Retainer Work [UI_Operation]

- An issue that the time values in market history were not converted into local time has been fixed.

### Better Market Board [UI_Optimization]

- An issue that Interrupt Hotkey on controllers may not work while skipping purchase confirmation has been fixed.

### Right-click Movement [General]

- The destination indicator feature has been refactored. Now the feature provides three different types `Markers Always`, `Fade-out Markers` and `No Marker`. The appearance and animation have also been refined.
- The movement logic has been refactored. Now the feature provides three different modes `Head Straight`, `Vnavmesh Navigation` and `Smart Hybrid`. The stability and general feeling have also been improved.
- The configuration option `Fixed Orientation While Moving` has been removed due to its weird character behaviours from others' perspective.
- An issue that the character may move even when there's no user input in some rare cases has been fixed.

### Better Frame Rate Cap [System]

- The frame rate cap in practical would be more accurate.

### Auto Ninjutsu [Action]

- The module now provides two modes:
    - Auto Mode: Automatically perform Mudras in order. It's configurable whether the module would also perform Ninjutsu after finishing all the Mudras.
    - Manual Mode: Allow each mudra to be performed only once to prevent misclicking.
