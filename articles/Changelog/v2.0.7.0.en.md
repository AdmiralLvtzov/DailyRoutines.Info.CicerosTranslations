## Framework Modifications

- A set of fuzzy search algorithms has been introduced for module searching.
- Now the "Amaurot" server in China data center, which is not open at the moment, has been excluded in all the server lists.

## New Modules

### Item Using Command [Assistance]

*Recommended Module*

Add a new command /pdr {0} <item ID/item name/item name in pinyin> to use a specific item in the inventory.

![UseItemCommand](/assets/Changelog/2.0.7.0/UseItemCommand.png)

### Better Inn Access [System]

*Recommended Module*

Remove confirmation while entering or leaving Inn and provide a command /pdr inn for entering Inn from anywhere except in duty.

![FastEnterAndLeaveInn](/assets/Changelog/2.0.7.0/FastEnterAndLeaveInn.png)

### Set Try-on In Player Inspection [UI_Operation]

*Recommended Module*

Introduce a new button in the player inspection interface, allowing to try on the appearance set of the inspected character.

![AutoTryOnPlayerOutfit](/assets/Changelog/2.0.7.0/AutoTryOnPlayerOutfit.png)

![AutoTryOnPlayerOutfit-UI](/assets/Changelog/2.0.7.0/AutoTryOnPlayerOutfit-UI.png)

## Module Modifications

### Healer Helper [Action]

- The job icon of the target would be displayed in the notification of action redirection.

![HealerHelper-Notification](/assets/Changelog/2.0.7.0/HealerHelper-Notification.png)

### Auto Skip Cutscenes [System]

- The animation after logging in at Inn could be skipped now.

### Quickly Exchange Seals For Items [UI_Operation]

- An issue that, the logic could still be executed when the quantity was not set or was set to 0, has been fixed.

### Better Duty Finder Settings [UI_Optimization]

- An issue that the button order may vary has been fixed.

### Better Market Board [UI_Optimization]

- The efficiency of acquiring market data in the Item Help interface has been greatly improved.

### Auto Block Status Effect [Combat]

- An issue that the module didn't work has been fixed.
- An issue that, the character orientation would be fixed after successfully blocking any status, has been fixed.
- The implementation has been streamlined.

### Auto Restore Furniture [UI_Operation]

- The module has been fully refactored.
    - The floating window is now created using vanilla assets instead of ImGui in the past.
    - `Failed to complete housing action.` exceptions which may be thrown multiple times during batch operations has been blocked.

### Auto Lucid Dreaming [Action]

- The module has been fully refactored.
- An issue that `Lucid Dreaming` couldn't be executed automatically when the current job is `Pictomancer` has been fixed.
- An issue that the module would work for jobs that shouldn't execute `Lucid Dreaming` in any case has been fixed.
- New configuration option `Send Chat Message`. Disabled by default.
- The notification appearance has been adjusted.
- The module description has been adjusted.

### Auto Hunt Mobs Finder [Script]

- The module logic has been refactored.

### Auto Gather In The Diadem [Script]

- The module logic has been fully refactored, adopting a more maintainable and streamlined architecture.

### Disable Background Game Rendering [System]

- The compatibility with the NamePlate interface has been improved.

### Auto Redirect Ground-Targeted Actions [Action]

- An issue that the game may stutter during the first redirection has been fixed.

### Better World Visit [System]

- The interaction logic of the component in Server Info Bar has been adjusted.
    - Left Click: Access the Better World Visit interface
    - Right Click: Return to Home World
- The drawing logic of the interface has been adjusted.

### Better Chat Panel [UI_Optimization]

- The stability of the interface has been improved.

### Better Mount Roulette [System]

- An issue of mixed use of selected mount displaying and searching has been fixed.

### Auto Display Available NPC Vendor [UI_Optimization]

- The module description has been adjusted.
- The name of the right-click menu has been adjusted.
- New features have been introduced, which allow users to inquire purchase and usage information at NPC vendors.

![AutoShowItemNPCShopInfo-Destination](/assets/Changelog/2.0.7.0/AutoShowItemNPCShopInfo-Destination.png)

- The style and layout of the item usage interface have been redesigned to match those of the item purchase interface.

![AutoShowItemNPCShopInfo-Source](/assets/Changelog/2.0.7.0/AutoShowItemNPCShopInfo-Source.png)

### Auto Expert Delivery [UI_Operation]

- New configuration option `Ignore Ultimate Raid Items`, to ignore all the gears purchased from ultimate raid totems. Enabled by default.

### Better Crafting Log [UI_Optimization]

- New configuration option `Notify After Synthesis Is Complete`, to send a notification after quick synthesis is complete. Enabled by default.
- The logic of the `Higher Quick Synthesis Cap` feature has been adjusted.
    - Before the update, users could only perform a maximum of 255 quick synthesis per operation.
    - Now users could set any workload. The workload would be batched if the workload exceeds 255.
- New button `x synthesis` in the recipe `Preview` interface, to repeat the synthesis x times following the current synthesis solution.
