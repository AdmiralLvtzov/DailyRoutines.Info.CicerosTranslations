## Framework Modifications

- The tray notification chain has been refactored.
- The module management logic has been improved.
- An issue that the module cards may extend beyond the right border has been fixed.
- A large amount of module categories have been adjusted. (The following content have no impact on user experience, just for developer reference)
    - Categories Removed:
        - `UI Operation` (`UIOperation`)
        - `UI Optimization` (`UIOptimization`)
    - New Category:
        - `Interface` (`Interface`)
    - Internal Name Adjustment:
        - `Notification`: `Notice` → `Notification`
    - New `Tier-1 Category` and `Tier-2 Category` have been introduced. `Tier-1 Category` is for general purposes while `Tier-2 Category` is for dedicated purposes. A module would belong to `Tier-2 Category` if it's assigned to both. A module would belong to the category appears later in the order if it could be assigned to multiple `Tier-1` and `Tier-2` categories. Refer to the following for details:
    - `Tier-1 Category`:
        - General
        - Notification
        - System
        - Interface
        - Combat
        - Script
        - Assistance
    - `Tier-2 Category`:
        - Duty
        - Recruitment
        - Action
        - Craft & Gather
        - Gold Saucer
    - All the modules belonged to `UI_Operation` and `UI_Optimization` in the past have been assigned to `Interface`. Refer to the following section `Module Category Adjustments` for other adjustments.

## Module Modifications

### Auto Track Players [System]

- The logic has been fully refactored.
- Now the feature and history of `Permanent Track` have been removed. `Temporary Track` remains, and the history would be cleared after switching zones.
- New interface in vanilla game style for assigning Waymarks to targets, as well as targeting or removing entries in the list.

![AutoTrackPlayers-Addon](/assets/Changelog/2.0.9.0/AutoTrackPlayers-Addon.png)

### Auto Adjust Signs [Combat]

- The module is now a Recommended Module.
- The logic has been refactored.

### Duty Joining & Leaving Command [Assistance]

- The module no longer requires authentication.
- The module is now a Recommended Module.

### Reset All Striking Dummy [Combat]

- The module is now a Recommended Module.

### Auto Remove Login Cooldown [System]

- The logic while queuing has been improved. Now the dialog box would no longer appear and disappear at a high frequency.

### Auto Login [General]

- The logic of `Login Once` has been adjusted to logging in once in the current process, which would not be effected by the module lifecycle.
- The configuration has been refactored. Now it adopts a match combination of "World and Character". It's required to re-configure the module after the update.
    - `World` now suggests a different meaning. It was referring to the World of a character index, now it's referring to Home World of a character name.
- The parameters have been adjusted accordingly. Some existing commands may require slight adjustments after the update.
    - Before the update, the command only took a character index. Now it also takes a character name.
    - The first parameter is a required `character index/character name`.
    - The second parameter is an optional `world name/world ID`.
- The logic has been fully refactored. Now it no longer performs multiple attempts based on information acquired. Instead, it directly navigates to the character matches the condition most and initiates a login process. The module also supports logging in to a Voyager character now.

### Better World Visit [System]

- Now the module would hold its tasks until all the tasks belong to `Auto Login` has finished while logging out.
- Now the module would hold its tasks until the character has actually logged out.

### Signs In Party List [Combat]

- An issue that the module may try to modify     certain UI nodes in Party List even when Party List doesn't exist has been fixed.

### Bait Switching Command [Assistance]

- Now fish baits in `Cosmic Exploration` are supported.

### Auto Store To Armoire [Interface]

- An issue that the module would try to store an item to Armoire even when the item already exists in Armoire has been fixed.
- The efficiency while storing items has been greatly improved.
- An issue that item names were missing in module logs has been fixed.
- Now the floating window is created using vanilla game assets.
