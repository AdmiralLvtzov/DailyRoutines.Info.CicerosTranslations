## 본체 변경

- 트레이 알림 트리거 체인 재구축
- 모듈 관리 로직 최적화
- 설정 UI의 카드 컴포넌트가 오른쪽 경계를 넘어 그려지던 문제 수정
- 모듈 분류 조정 (아래 내용은 일반 사용자와 무관하며, 내부 분류 기준 변경 안내입니다)
    - 삭제된 분류:
        - `UI 조작` (`UIOperation`)
        - `UI 개선` (`UIOptimization`)
    - 신규 분류:
        - `인터페이스` (`Interface`)
    - 분류 내부명 조정:
        - `알림 기능`: `Notice` → `Notification`
    - `일차 분류`와 `이차 분류`로 구분. `일차 분류`는 포괄적이고, `이차 분류`는 구체적입니다. 모듈이 `일차 분류`와 `이차 분류` 모두에 해당할 경우 해당 `이차 분류`로 분류됩니다. 모듈이 여러 `일차 분류` 또는 `이차 분류`에 해당할 경우 표시 순서가 더 뒤인 분류가 우선됩니다 (자세한 내용은 아래 참조)
    - `일차 분류`에 포함:
        - 일반
        - 알림 기능
        - 시스템
        - 인터페이스
        - 전투
        - 스크립트
        - 도우미
    - `이차 분류`에 포함:
        - 콘텐츠
        - 파티 모집
        - 액션
        - 제작 및 채집
        - 골드 소서
    - 기존 `UI 조작` 및 `UI 개선` 모듈 분류는 일괄적으로 `인터페이스`로 변경됩니다. 기타 모듈 분류 변경은 아래 `모듈 분류 조정`을 참조하세요
- 플러그인 생애주기 재구축, 전면 비동기 초기화 및 비동기 언로드로 변경
- 메인 UI가 열린 상태에서 플러그인을 언로드하면 null 참조 예외가 발생할 수 있던 문제 수정
- 플러그인 언로드 중에도 주변 플레이어 정보를 계속 가져와 null 참조 예외가 발생하던 문제 수정
- 플러그인 내부 네트워크 연결 로직 대폭 최적화
- 인증 노드를 모두 사용할 수 없을 때 플러그인 로드로 인해 게임이 복구 가능한 무응답 상태에 빠지던 문제 수정

- 게임 네이티브 UI 추가. 각 플레이어에 대상 표식을 할당할 수 있으며, UI 내에서 제거, 선택 등의 작업 가능

- 추천 모듈로 변경
- 모듈 구현 로직 재구축

### 던전 1회 자동 입장/퇴장 [도우미]

- 온라인 인증 요구사항 제거

- 추천 모듈로 변경

### 로그아웃 조작시 쿨다운 잠금 해제 [시스템]

- 대기열 대기 시 처리 로직 최적화, 다이얼로그가 반복적으로 나타났다 사라지는 문제 방지

### 자동 로그인 [일반]

- `한 번 로그인` 모드 로직 변경: 현재 프로세스에서 한 번만 로그인하며, 모듈 생애주기의 영향을 받지 않음
- 설정 재구축, `서버+캐릭터 이름` 매칭 방식으로 변경. 재설정 필요
    - `서버` 의미 변경. 기존에는 캐릭터 인덱스에 해당하는 현재 서버였으나, 이제는 캐릭터 이름에 해당하는 고향 서버
- 설정 변경에 따라 명령어 파라미터도 변경. 기존 명령어 조정이 필요할 수 있음
    - 기존에는 플레이어 캐릭터 인덱스만 지원했으나, 새로이 플레이어 캐릭터 이름도 지원
    - 첫 번째 파라미터는 `캐릭터 인덱스/캐릭터 이름`으로 고정
    - 두 번째 파라미터는 옵션인 `서버 이름/서버 ID`
- 내부 로직 완전 재구축. 서버 정보를 순회하거나 여러 번 재시도하지 않고, 가장 적합한 로그인 가능 캐릭터에 직접 로그인하며, 데이터 센터 여행 중인 캐릭터에도 로그인 지원

### 빠른 서버 텔레포트 명령어 [시스템]

- 로그아웃 실행 시 `자동 로그인` 모듈 관련 로직이 종료된 후에 후속 로직을 실행하도록 보장
- 로그아웃 실행 시 캐릭터가 로그아웃된 후에 후속 로직을 실행하도록 보장

- 이미 추억의 보관함에 있는 아이템을 보관하려 시도하던 문제 수정
