## Framework Modifications

- The appearance of the update info interface has been improved.
- The logic of enabling and disabling Daily Routines have been improved.
- An issue that the `Skip Logos` feature didn't work due to initialization order has been fixed.
- The logic of determining whether an object nearby is a player has been improved.
- Now travelling via map connections would be considered as a potential option while navigating from a map to another.

## Module Information Adjustments

### Praetorium Helper [Duty]

- New internal name is `PraetoriumHelper`. It's required to re-enable and re-configure the module after the update.

### Alexander Cuff Of Father Helper [Duty]

- New internal name is `CuffOfTheFatherHelper`. It's required to re-enable and re-configure the module after the update.

## New Modules

### Simplified System Menu [Interface]

*Recommended Module*

Provide a simplified System Menu excluding the options unrelated to daily game experience for a neat layout.

![SimplifiedSystemMenu](/assets/Changelog/2.1.0.0/SimplifiedSystemMenu.png)

### Tell In Duty [Duty]

*Recommended Module*

Allow to send and receive /tell in duty.

### Pool Of Tribute Helper [Duty]

*Recommended Module*

Automatically mark the party member with the status "Fetters" in order to help determine the correct "ama-no-iwato" in the duty "Pool Of Tribute".

### Better Sound Effect Parameters [System]

*Recommended Module*

Allow system sound effects (<se.x>) to be played on all the channels and to be repeated more than three times.

### Better Enmity List [Combat]

- The logic has been streamlined and the performance has been improved.
- New configuration option `Always Unlock`. Enmity List could be locked during certain mechanisms in order to prevent targeting enemies. For example, during "Manusya Confuse" in Boss Combat 2 "Sanduruva" in the duty "Tower of Zot". Enable the option for a always unlocked Enmity List. Enabled by default.
- The resource management has been improved.
- Now `Auto Display Hidden Casts` is a recommended Module. Similar feature within the module has been removed.
- The appearance has been redesigned.
    - The enmity display has been removed due to limited content distribution from the server.
    - The weight of HP percentage has been increased.
    - The customization of the text format and the background transparency has been removed.
    - Now HP percentage is fixed on the left while status effects are on the right.

    ![OptimizedEnemyList-Interface](/assets/Changelog/2.1.0.0/OptimizedEnemyList-Interface.png)

### Better Teleportation [Interface]

- New smart search memory system has been introduced for better interaction.

### Sound Effect Throttler [System]

- The configuration option `Volume` has been removed, since the module now works on volume control of all the system sound effects.

### Auto FATE [Script]

- The logic has been fully refactored and streamlined.
- Now module logs would be printed.
- The logic of navigating to the closest enemy in combat has been removed. Now the module passes the task to BossMod (Reborn).
- Now the module would prioritize collecting items in collection FATEs.
- Now `Ignore Action Target Obstruction Status` is a conflict module.
- Now `Auto Summon Companion` is a recommended module.
- Now the module would initiate a wayfinding process while navigating to a Gemstone Trader.
- Now the module would estimate the Time-To-Kill (TTK) of `Forlorn Maiden` / `Forlorn` before engaging. If the TTK is longer than the FATE remaining time, `Forlorn Maiden` / `Forlorn` would be ignored.
- Now the module would try to navigate to the nearest safe zone after joining & leaving a duty for combat disengagement.
- An issue that non-FATE targets may be assigned a higher priority compared with FATE targets in some cases has been fixed.
- The drawing logic of the configuration interface has been refactored.
- New configuration guides regarding prerequisite modules and plugins has been introduced.

### Auto Refuse Trade Request [System]

- New configuration option `Delay`, to introduce a given delay before refusing a trade.
- Extra text commands now support the `/wait` command and the `<wait>` parameter.

### Auto Enable Attack [Combat]

- The logic has been fully refactored adopting a new underlying approach.

### Field Operation Entry Command [Assistance]

- The logic has been greatly streamlined.
- Now the module supports entering another field via the commands in duty.

### Auto Hunt Mob Finder [Script]

- An issue that scan results couldn't be saved properly has been fixed.

### Leave Recruitment Then Join Another [Recruitment]

- The `Leave Then Join` button would be available only when in a recruitment or a cross-world party.

### Auto Display Hidden Casts [Combat]

- More UI logic has been overrided.
